# 一、设计目标

每个队友都应该满足：

```Plain
有实力
有性格
有成长
有故事
```

而不是：

```Plain
一个数字
```

---

# 二、生成逻辑总览

队友生成分为五层：

```Plain
基础信息
↓
五维属性
↓
选手原型
↓
人格特征
↓
关系倾向
```

---

最终形成：

```Plain
电竞选手NPC
```

# 队友生成最终公式

```Plain
位置
+
年龄
+
五维属性
+
潜力
+
选手原型
+
人格标签
+
关系倾向
=
完整队友NPC
```

这样生成出来的队友才不会是：

```Plain
ADC 82
```

而会是：

```Plain
18岁ADC

天赋95

卷王

社恐

未来传奇

每天训练到凌晨三点
```

玩家会记住这个人，并在几年后看到他成长为队友、竞争对手、宿敌或接班人。这个系统后面还能直接支撑：

```Plain
队友系统
转会系统
宿敌系统
传承系统
AI选手生态系统
```

所以它实际上是整个电竞世界NPC生态的基础层。
