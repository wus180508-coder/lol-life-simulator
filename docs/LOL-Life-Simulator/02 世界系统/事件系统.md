
# 一、系统定位

## 系统名称

事件系统（Event Engine）

---

## 系统定位

事件系统是《竞男人生模拟器》的核心内容系统。

玩家体验到的电竞人生并不是由数值构成，而是由一个个人生事件构成。

数值负责决定：

```Plain
发生什么
```

事件负责呈现：

```Plain
为什么发生
```

---

例如：

玩家不会记得：

```Plain
游戏理解+3
```

但会记得：

```Plain
你连续熬夜Rank三个月

终于冲上韩服前100
```

---

因此：

```Plain
内容量 = 事件量
```

事件系统将成为整个游戏的主要内容来源。

# 五、事件触发机制

事件并非完全随机。

采用：

```Plain
随机权重
+
条件筛选
```

---

## 基础公式

```Plain
最终权重

=

事件基础权重

×

属性修正

×

阶段修正

×

事件链修正
```

---

## 示例

事件：

```Plain
撞车职业选手
```

条件：

```Plain
游戏理解 ≥ 70

对线能力 ≥ 75
```

满足后进入事件池。

---

# 六、事件链系统（新增）

## 系统定位

多个事件组成连续剧情。

---

## 示例

家庭冲突事件链

---

第一步

```Plain
偷跑网吧
```

↓

第二步

```Plain
被父母发现
```

↓

第三步

```Plain
电脑被没收
```

↓

第四步

```Plain
家庭会议
```

↓

第五步

```Plain
最终决定
```

---

结果：

```Plain
获得支持

或

被严格限制
```

---

## 事件链原则

单链长度：

```Plain
3~5个事件
```

---

每阶段至少：

```Plain
10条核心事件链
```

---
